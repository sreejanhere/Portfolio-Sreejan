Project Overview

Implemented an OLED display interface using SH1106 controller with Arduino, focusing on I2C communication protocol and display buffer management. This project enabled displaying custom text, graphics, and real-time data on a high-contrast OLED screen.

The interface supports various display modes including scrolling text, custom animations, sensor data visualization, and menu systems for embedded applications.

Technical Specifications

  • Display: SH1106 OLED (128x64 pixels)
  • Interface: I2C (Two-wire serial)
  • Controller: Arduino Uno/Nano
  • Communication: I2C at 400kHz
  • Library: Adafruit_SSD1306 & Adafruit_GFX
  • Power: 3.3V or 5V operation
  • Resolution: 128x64 monochrome
  • Address: 0x3C (default I2C address)

Components Used

  • SH1106 OLED Display Module
  • Arduino Uno/Nano
  • I2C Pull-up Resistors (4.7kŒ©)
  • Breadboard and Jumper Wires
  • USB Cable for programming

I2C Communication Protocol

The SH1106 OLED display uses I2C protocol for communication:

Step 1: Start Condition

Master initiates communication with start bit

Step 2: Address Frame

Send device address (0x3C) with write bit

Step 3: Control Byte

Send command/data selection byte

Step 4: Data Transfer

Send display data or commands

Quick Firmware Example (concept)

Verified and Compiled Arduino IDE Code 

#include 
#include 

// Correct for 128x64 SH1106 OLED
U8G2_SH1106_128X64_NONAME_F_HW_I2C display(U8G2_R0, U8X8_PIN_NONE);

void setup() {
  display.begin();
}

void loop() {
  display.clearBuffer();

  // Big spooky font
  display.setFont(u8g2_font_ncenB12_tr);

  // Center text manually
  display.drawStr(22, 26, "HAPPY");
  display.drawStr(2, 56, "HALLOWEEN");
// displays HAPPY HALLOWEEN in two consecutive lines 
  display.sendBuffer();
  delay(100);
}
