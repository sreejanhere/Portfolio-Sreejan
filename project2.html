ESP32 — Dual LED Control (MIT App Inventor)

Status: Completed✅

A beginner-friendly IoT control demo: Two LEDs (GPIO2 & GPIO4) controlled by a self-made Android app built in MIT App Inventor via Bluetooth.

Project Overview

This project demonstrates controlling two LEDs connected to an ESP32 development board using a custom Android interface created in MIT App Inventor. The mobile app sends Bluetooth serial messages that the ESP32 reads and maps to the two digital outputs (GPIO2 and GPIO4).

Purpose: learn wireless serial communication, basic GPIO control, and simple mobile UI design for hardware control.

Technical Specifications

  • Microcontroller: ESP32-WROVER-E development board
  • Control method: Bluetooth serial (paired phone + MIT App Inventor)
  • LED pins: GPIO2 (LED 1) and GPIO4 (LED 2)
  • Programming: Arduino IDE (ESP32 core)
  • Power source: USB 5V or regulated supply

Components Used

  • ESP32 development board (WROVER-E)
  • Two 3mm LEDs and 220Ω resistors
  • Breadboard, jumper wires, USB cable
  • Android phone running the MIT App Inventor-built app

How it works (high level)

  • User opens the custom Android app (created with MIT App Inventor) and pairs with the ESP32 via Bluetooth.
  • App contains simple ON / OFF buttons for LED 1 and LED 2. Pressing a button sends a short text message to the ESP32 over Bluetooth serial.
  • ESP32 receives the message, toggles the appropriate GPIO pin, and optionally sends a small acknowledgment back to the app for UX feedback.

Note: this page intentionally avoids listing specific command characters — the app sends simple button-driven messages and the ESP32 maps them to LED actions.

Learning Outcomes

  • Built a full phone-to-hardware control flow (App Inventor → Bluetooth → ESP32)
  • Practiced ESP32 GPIO control and debouncing techniques
  • Used Arduino IDE for quick prototyping and Serial Monitor for debugging
  • Gained experience designing a basic, usable UI in MIT App Inventor

Challenges & Solutions

Challenge: Ensuring stable Bluetooth pairing across phones.

Solution: Verified device name, closed other Bluetooth clients while testing, and used short, single-action button calls in the app for reliability.

Challenge: Accidental multi-character sends from the app.

Solution: Designed app buttons to send focused messages and added minimal input checks in the ESP32 firmware.

Future Enhancements

  • Visual ACKs inside the app for better UX (show ON/OFF state)
  • Brightness control using PWM (send numeric values)
  • Web / Wi-Fi control integration for remote access
  • Publish a small tutorial + downloadable package (code + schematic)
